Pixel Agents for Codex is a VS Code extension that turns multi-agent coding into something you can actually see and manage. Instead of treating AI sessions as invisible terminal processes, it represents each Codex session as a character inside a pixel-art office. Every agent is backed by a real, stateful Codex session, so the UI is not just decorative. It reflects live session activity such as tool usage, waiting states, permission requests, task ownership, and workspace context. This makes it easier to understand what each agent is doing, where it is blocked, and how work is progressing across the team.

The extension also adds a Mission Control layer on top of those sessions. Mission Control tracks structured runtime state for agents, tasks, approvals, events, artifacts, token usage, and workspace assignments. That gives users a higher-level operational view of the system, not just animated characters on a screen. You can inspect individual agents, monitor approvals, follow task progress, and keep track of which session is responsible for which part of the work. The goal is to make parallel AI execution easier to supervise, reason about, and debug inside the editor.

A major part of the project is its master orchestration workflow. A top-level master agent takes a high-level task, plans a clean split of the work, provisions isolated git worktrees, and launches separate Codex worker sessions to execute those assignments in parallel. Each worker gets a scoped area of ownership, its own prompt, and its own workspace so the overall run stays organized. Once the workers finish, the extension supports review of their changes before those results are applied back into the main checkout. In practice, this makes Pixel Agents for Codex both a visualization layer and a coordination system for structured, stateful, multi-session software development inside VS Code.
